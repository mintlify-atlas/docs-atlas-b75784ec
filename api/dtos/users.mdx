## Overview

Users DTOs handle user registration, authentication, profile updates, and account recovery in the Tonight Pass SDK. These include comprehensive validation for user data and secure password requirements.

---

## CreateUserDto

Used to register a new user account.

### Class Definition

```typescript
export class CreateUserDto {
  identifier: CreateUserIdentifierDto;
  identity: CreateUserIdentityDto;
  password: string;
}
```

### Properties

<ParamField path="identifier" type="CreateUserIdentifierDto" required>
  The user's identifier information (email, phone, username).
</ParamField>

<ParamField path="identity" type="CreateUserIdentityDto" required>
  The user's identity information (name, gender, birth date, etc.).
</ParamField>

<ParamField path="password" type="string" required>
  The user's password.
  
  **Validation:**
  - Must be at least 8 characters long
  - Must contain at least one uppercase letter
  - Must contain at least one lowercase letter
  - Must be secure (additional password complexity requirements)
</ParamField>

### Example

```typescript
const createUser: CreateUserDto = {
  identifier: {
    email: "alice@example.com",
    username: "alice"
  },
  identity: {
    firstName: "Alice",
    lastName: "Smith",
    gender: UserIdentityGender.Female,
    birthDate: new Date("1990-01-01")
  },
  password: "SecurePassword123"
};
```

## CreateUserIdentityDto

Used to define a user's identity during registration.

### Class Definition

```typescript
export class CreateUserIdentityDto {
  firstName: string;
  lastName: string;
  gender: UserIdentityGender;
  avatarUrl?: string;
  birthDate: Date;
  links?: string[];
}
```

### Properties

<ParamField path="firstName" type="string" required>
  The user's first name.
  
  **Validation:**
  - Length: 2-32 characters
  - Must be composed of letters only
</ParamField>

<ParamField path="lastName" type="string" required>
  The user's last name.
  
  **Validation:**
  - Length: 2-32 characters
  - Must be composed of letters only
</ParamField>

<ParamField path="gender" type="UserIdentityGender" required>
  The user's gender.
  
  **Valid values:**
  - `UserIdentityGender.Male`
  - `UserIdentityGender.Female`
  - `UserIdentityGender.NonBinary`
</ParamField>

<ParamField path="avatarUrl" type="string">
  URL to the user's avatar image.
  
  **Validation:**
  - Must match the user avatar URL regex pattern for creation
</ParamField>

<ParamField path="birthDate" type="Date" required>
  The user's date of birth.
  
  **Validation:**
  - Must be a valid date
</ParamField>

<ParamField path="links" type="string[]">
  Array of links to the user's social media or website.
  
  **Validation:**
  - Maximum 5 links
  - Each must be a valid URL
</ParamField>

  - Each must be a valid URL
