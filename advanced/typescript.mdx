The Tonight Pass SDK is built with TypeScript-first design, providing comprehensive type safety and excellent developer experience with full IntelliSense support.

## Type Safety

The SDK uses advanced TypeScript features to ensure type safety across all API calls:

<CodeGroup>
```typescript Node.js
import { TonightPass } from "tonightpass";

const tnp = new TonightPass({
  baseURL: "https://api.tonightpass.com",
  apiKey: "your-api-key"
});

// TypeScript infers the response type automatically
const user = await tnp.users.get("username");
// user is typed as User
```

```typescript React
import { useUser } from "tonightpass/react";

function UserProfile() {
  const { data: user, error, isLoading } = useUser("username");
  // user is typed as User | undefined
  
  if (isLoading) return <div>Loading...</div>;
  if (error) return <div>Error: {error.message}</div>;
  
  return <div>{user?.identity.fullName}</div>;
}
```
</CodeGroup>

## Core Types

### API Response Types

All API responses follow a consistent structure:

```typescript packages/node/src/rest/client.ts
export type SuccessfulAPIResponse<T> = {
  success: true;
  data: T;
};

export type ErroredAPIResponse = {
  success: false;
  message: string;
  statusCode: number;
  errors?: {
    [key: string]: string[];
  };
};

export type APIResponse<T> = SuccessfulAPIResponse<T> | ErroredAPIResponse;
```

<Note>
  The SDK automatically throws `TonightPassAPIError` for failed requests, so you only need to handle successful responses in your code.
</Note>

### Generic Type Helpers

The SDK exports several utility types for working with API endpoints:

<Accordion title="PathsFor<Method>" icon="route">
  Extract all valid paths for a specific HTTP method:

  ```typescript packages/node/src/rest/client.ts
  export type PathsFor<M extends Options["method"]> = Extract<
    Endpoints,
    { method: M }
  >["path"];
  ```

  **Usage:**
  ```typescript
  import type { PathsFor } from "tonightpass";

  // Get all GET endpoint paths
  type GetPaths = PathsFor<"GET">;
  // "/users/@:username" | "/organizations/@:organizationSlug" | ...
  ```
</Accordion>

<Accordion title="ResponseFor<Method, Path>" icon="arrow-turn-down-left">
  Get the response type for a specific endpoint:

  ```typescript packages/node/src/rest/client.ts
  export type ResponseFor<
    M extends Options["method"],
    P extends PathsFor<M>,
  > = Extract<Endpoints, { method: M; path: P }>["res"];
  ```

  **Usage:**
  ```typescript
  import type { ResponseFor } from "tonightpass";

  type UserResponse = ResponseFor<"GET", "/users/@:username">;
  // User
  ```
</Accordion>

<Accordion title="Body<Method, Path>" icon="file-code">
  Get the request body type for a specific endpoint:

  ```typescript packages/node/src/rest/client.ts
  export type Body<M extends Options["method"], P extends PathsFor<M>> = Extract<
    Endpoints,
    { method: M; path: P }
  >["body"];
  ```

  **Usage:**
  ```typescript
  import type { Body } from "tonightpass";

  type CreateUserBody = Body<"POST", "/users">;
  // CreateUserDto
  ```
</Accordion>

<Accordion title="Response<Method, Path>" icon="brackets-curly">
  Get the full API response (including success/error wrapper):

  ```typescript packages/node/src/rest/client.ts
  export type Response<
    M extends Options["method"],
    P extends PathsFor<M>,
  > = APIResponse<Extract<Endpoints, { method: M; path: P }>["res"]>;
  ```
</Accordion>

## Error Handling

### TonightPassAPIError

The SDK provides a typed error class for API errors:

```typescript packages/node/src/rest/client.ts
export class TonightPassAPIError<T> extends Error {
  public readonly status: number;

  constructor(
    public readonly response: RedaxiosResponse<APIResponse<T>>,
    public readonly data: ErroredAPIResponse
  ) {
    super(data.message);
    this.status = response.status;
  }
}
```

**Usage:**

```typescript
import { TonightPassAPIError } from "tonightpass";

try {
  const user = await tnp.users.get("invalid-username");
} catch (error) {
  if (error instanceof TonightPassAPIError) {
    console.error(`API Error ${error.status}: ${error.message}`);
    
    // Access validation errors
    if (error.data.errors) {
      Object.entries(error.data.errors).forEach(([field, messages]) => {
        console.error(`${field}: ${messages.join(", ")}`);
      });
    }
  }
}
```

## Client Configuration

### ClientOptions Type

```typescript packages/node/src/rest/client.ts
export type ClientOptions = {
  readonly baseURL: string;
  readonly apiKey?: string;
  readonly accessToken?: string;
  readonly cache?: CacheOptions;
};
```

<Accordion title="Cache Options">
  ```typescript
  export type CacheOptions = {
    enabled: boolean;
    ttl?: number; // Time to live in milliseconds
    methods?: Array<"GET" | "POST" | "PUT" | "PATCH" | "DELETE">;
  };
  ```

  **Example:**
  ```typescript
  const tnp = new TonightPass({
    baseURL: "https://api.tonightpass.com",
    apiKey: "your-api-key",
    cache: {
      enabled: true,
      ttl: 60000, // 1 minute
      methods: ["GET"] // Only cache GET requests
    }
  });
  ```
</Accordion>

## Best Practices

<CardGroup cols={2}>
  <Card title="Import Types" icon="file-import">
    Always import types for better IDE support:
    ```typescript
    import type { User, Organization } from "tonightpass";
    ```
  </Card>

  <Card title="Use Type Guards" icon="shield-check">
    Check response success before accessing data:
    ```typescript
    const response = await tnp.users.get("username");
    if (response.success) {
      console.log(response.data);
    }
    ```
  </Card>

  <Card title="Leverage Inference" icon="wand-magic-sparkles">
    Let TypeScript infer types when possible:
    ```typescript
    // No need to manually type
    const user = await tnp.users.get("username");
    ```
  </Card>

  <Card title="Handle Errors" icon="triangle-exclamation">
    Always wrap API calls in try-catch:
    ```typescript
    try {
      const user = await tnp.users.get("username");
    } catch (error) {
      // Handle error
    }
    ```
  </Card>
</CardGroup>

<Warning>
  The SDK requires TypeScript 5.0 or higher for optimal type inference and generic support.
</Warning>
