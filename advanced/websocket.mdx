The Tonight Pass SDK provides WebSocket support for real-time updates on channels, messages, and other events.

## Overview

WebSocket connections enable real-time bidirectional communication for:

- Real-time chat messages
- Typing indicators
- Channel updates
- Member join/leave events
- Live notifications

<Note>
  WebSocket support was added in version 0.0.180 with cross-platform compatibility for Node.js, browsers, and React Native.
</Note>

## Getting Started

### Basic Connection

<CodeGroup>
```typescript Node.js
import { channelsWS } from "tonightpass/ws";

const ws = channelsWS({
  baseURL: "wss://api.tonightpass.com",
  debug: true,
  maxReconnectAttempts: 5,
  reconnectInterval: 2000
});

// Connect to a specific channel
await ws.connect("channel-id", "your-access-token");

// Listen for new messages
ws.onMessageCreate((event) => {
  console.log("New message:", event.data);
});
```

```typescript React
import { useWebSocket } from "tonightpass/react";

function ChatComponent({ channelId }: { channelId: string }) {
  const { data: messages, subscribe } = useWebSocket(
    `/channels/~me/:channelId/ws`,
    { channelId, token: "your-access-token" }
  );

  useEffect(() => {
    const unsubscribe = subscribe("channel_message_create", (event) => {
      console.log("New message:", event.data);
    });

    return unsubscribe;
  }, [subscribe]);

  return <div>{/* Render messages */}</div>;
}
```
</CodeGroup>

## WebSocket Client

### Configuration Options

```typescript packages/node/src/ws/types/index.ts
export type WebSocketClientOptions = {
  baseURL: string;
  reconnectInterval?: number;
  maxReconnectAttempts?: number;
  debug?: boolean;
};
```

<ParamField path="baseURL" type="string" required>
  WebSocket server URL (e.g., `wss://api.tonightpass.com`)
</ParamField>

<ParamField path="reconnectInterval" type="number" default="1000">
  Milliseconds to wait between reconnection attempts
</ParamField>

<ParamField path="maxReconnectAttempts" type="number" default="3">
  Maximum number of reconnection attempts before giving up
</ParamField>

<ParamField path="debug" type="boolean" default="false">
  Enable debug logging for connection events
</ParamField>

## Available Endpoints

The SDK supports typed WebSocket paths:

```typescript packages/node/src/ws/endpoints.ts
export type WebSocketEndpoints =
  | WebSocketEndpoint<"/channels/~me/ws", WebSocketConnectOptions>
  | WebSocketEndpoint<
      "/channels/~me/:channelId/ws",
      WebSocketConnectOptions & { channelId: string }
    >
  | WebSocketEndpoint<
      "/channels/:organizationSlug/ws",
      WebSocketConnectOptions & { organizationSlug: string }
    >
  | WebSocketEndpoint<
      "/channels/:organizationSlug/:channelId/ws",
      WebSocketConnectOptions & { organizationSlug: string; channelId: string }
    >;
```

## Advanced Features

### Auto-Reconnection

The client automatically attempts to reconnect when the connection is lost:

```typescript packages/node/src/ws/client/index.ts
private handleReconnect<P extends WebSocketPaths>(
  path: P,
  options: WebSocketOptionsFor<P>
) {
  if (this.reconnectAttempts >= this.options.maxReconnectAttempts!) {
    this.log("Max reconnect attempts reached");
    return;
  }

  this.isReconnecting = true;
  this.reconnectAttempts++;

  this.reconnectTimer = setTimeout(() => {
    this.connect(path, options).catch((error) => {
      this.log("Reconnection failed:", error);
    });
  }, this.options.reconnectInterval);
}
```

**Monitor reconnection status:**

```typescript
console.log("Connected:", ws.connected);
console.log("Reconnecting:", ws.reconnecting);
```

### Typing Indicators

Send typing indicators to other users:

```typescript
// Start typing indicator
ws.startTyping("channel-id");

// Stop typing indicator
ws.stopTyping("channel-id");
```

### Channel Subscriptions

Dynamically subscribe/unsubscribe from channels:

```typescript
// Subscribe to a channel
ws.subscribeToChannel("new-channel-id");

// Unsubscribe from a channel
ws.unsubscribeFromChannel("old-channel-id");
```

### Wildcard Event Listener

Listen to all events:

```typescript
ws.onAny((event) => {
  console.log("Event received:", event.type, event.data);
});
```

## Authentication

<Warning>
  WebSocket authentication uses subprotocols instead of headers for better compatibility with browsers and React Native.
</Warning>

```typescript packages/node/src/ws/client/index.ts
// Pass token via subprotocol if present (cleaner than query params)
const protocols: string[] = [];
if ("token" in options && typeof options.token === "string") {
  protocols.push(`access_token.${options.token}`);
}

this.ws =
  protocols.length > 0
    ? new WebSocket(url, protocols)
    : new WebSocket(url);
```

**Usage:**

```typescript
const ws = channelsWS();
await ws.connect("channel-id", "your-access-token");
```

## Cleanup

Always disconnect when done:

```typescript
// Disconnect and clean up
ws.disconnect();
```

<Note>
  In React, use cleanup functions in `useEffect` to automatically disconnect when components unmount.
</Note>
